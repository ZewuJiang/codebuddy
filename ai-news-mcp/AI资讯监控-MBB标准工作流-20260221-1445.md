## 🎯 核心标准

### **MBB 设计原则**

1. **金字塔原理** - 结论先行，论点→事实→影响→建议
2. **MECE 框架** - 互相独立、完全穷尽的分类
3. **So What 分析** - 每个洞察必须回答"那又怎样？"
4. **量化数据** - 用具体数字和百分比增强说服力
5. **行动导向** - 每项洞察必须指向具体行动

### **视觉标准**

- **配色**: 深海军蓝 (#1a3a52) + 珊瑚红 (#e63946) + 活力橙 (#f77f00)
- **字体**: STSong-Light / PingFang（中文完美支持）
- **布局**: 卡片式设计，圆角 6px，左侧色条标识优先级
- **间距**: 充足留白，标题区间距 40px，卡片间距 12px，时间线节点间距 55px

### **2. 内容分析（MECE 框架）**

**分类维度**:

| 类别 | 定义 | 优先级判断标准 |
|------|------|---------------|
| **组织变革** | 高管人事、组织架构调整 | 影响行业格局 → ⭐⭐⭐⭐⭐ |
| **伦理风险** | AI 伦理、品牌危机 | 品牌风险 > 40% → ⭐⭐⭐⭐ |
| **市场调整** | 工作室关闭、并购 | 涉及 5+ 家公司 → ⭐⭐⭐⭐ |
| **技术应用** | AI 工具使用、立场分化 | 影响开发流程 → ⭐⭐⭐ |
| **政策法规** | 合规、法律诉讼 | 成本增加 > 30% → ⭐⭐⭐ |

**分析模板**:
```
论点（Thesis）: 一句话核心观点
  ↓
事实（Facts）: 具体事件详情（200-250 字）
  ↓
影响（Impact）: So What 分析（短期/中期/长期 + 量化数据）
  ↓
建议（Action）: 具体行动（负责部门 + 完成期限 + 预算）
```

### **方式 2: 自动化（推荐用于日常）**

创建自动化脚本 `generate_daily_report.sh`:

```bash
#!/bin/bash

# AI 资讯监控自动报告生成脚本
# 每天早上 9:00 自动执行

DATE=$(date +%Y%m%d)
WORKDIR="/Users/zewujiang/Desktop/AICo/codebuddy"

cd $WORKDIR

# 1. 获取最新数据（MCP）
echo "📡 获取最新 AI 资讯..."
# 这里调用 MCP 获取数据并更新 Python 文件

# 2. 生成报告
echo "🎨 生成 MBB 报告..."
python3 create_mbb_deep_insight.py

# 3. 质量检查
echo "🔍 质量检查..."
if [ -f "AI战略资讯-深度洞察版-${DATE}.pdf" ]; then
    echo "✅ 报告生成成功: AI战略资讯-深度洞察版-${DATE}.pdf"
    open "AI战略资讯-深度洞察版-${DATE}.pdf"

    echo "❌ 报告生成失败"
    exit 1
fi

# 4. 归档旧报告（保留 30 天）
echo "📦 归档旧报告..."
find $WORKDIR -name "AI战略资讯-深度洞察版-*.pdf" -mtime +30 -exec mv {} archive/ \;

echo "🎉 完成！"
```

**设置定时任务**:

# 添加以下行（每天早上 9:00 执行）

## 🆘 故障排除

### **问题 1: 中文方块字**

**原因**: 字体未找到  
**解决**: 
```python
# 在代码中指定系统字体
font_name = 'STSong-Light'  # macOS
# 或
font_name = 'SimSun'  # Windows
```

---

### **问题 2: 文字溢出框外**

**原因**: 卡片高度计算不足  
**解决**:
```python
# 增加底部内边距
card_h = base_height + extra_height + 10
y_cursor -= 5  # 底部留白
```

---

### **问题 3: 元素重叠**

**原因**: 间距设置过小  
**解决**:
```python
# 标题与内容间距
self.y -= 40  # 从 35 增加到 40

# 时间线节点间距
y_pos = self.y - i * 55  # 从 50 增加到 55
```
