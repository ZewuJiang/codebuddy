## Roblox RBLX复盘（#49-#72）

### 涨跌幅数据严重事故（#49-#55，v7.1）

**核心教训**：1M/6M/1Y涨跌幅**三项全错**，源于：
1. 混淆"事件驱动跌幅"与"滚动区间涨跌幅"
2. AI依赖训练数据推测而非实时获取
3. 工作流缺少"必须从Finviz直接读取"的强制步骤
4. 写入后未交叉验证

**修补**：新增铁律#27 + 审计模式P11

### 最终审视17处优化（#56-#72，v8）

**核心教训**：EV计算过程不显示、毛利率影响基准年选错、Executive Summary缺保守中枢、FCF So What缺质量拆解、营收So What缺毛利润增速对比、Q4 KPI So What缺量化、跟踪指标表缺毛利率维度、Bull Case仅列标题缺论据、报告末尾缺核心矛盾提示。

### 多口径校准（#81-#83）
- SBC双口径未标注
- CapEx双口径混用（P&L 639亿 vs 现金流表767亿）
- D&A口径不一致

### 估值模型级联修正（#84-#85）
- D&A修正→EBITDA→EV/EBITDA级联
- SBC修正→DCF基年FCFF→全10年级联

**核心教训总结**：
1. D&A必须从年报附注获取，严禁EBITDA反推
2. SBC等Non-IFRS调整项必须从Reconciliation表提取
3. "估"标记不是终态，通过系统化搜索可削减至<5%
4. 估值模型参数修正必须级联更新所有依赖项
5. PDF提取需逻辑校验（毛利÷营收=毛利率）

## 审计模式库（P01-P17）

| 模式 | 名称 | 典型表现 | 检查方法 |
|------|------|---------|---------|
| P01 | 政策状态滞后 | 引用政策写"待实施"实际已生效 | 对每个政策做实时web_search验证 |
| P02 | 计算推导链缺失 | 给出"EV/EBITDA 7.4x"但不展示来源 | 搜索"≈""约""x"附近是否有推导 |
| P03 | 图表数据时间不同步 | 文本更新到最新季度，图表仍为上季 | 对比chart_data与文本表格季度 |
| P04 | 同比增速口径不一致 | 自算+17.4% vs 官方+17% | 搜索公司原始表述 |
| P05 | 三方数据差异未解释 | StockAnalysis 2.58x vs 手动算2.65x | 对比并标注差异来源 |
| P06 | 结构子章节遗漏 | 对照v7-v9结构模板逐一勾选 | 按第一轮审视清单检查 |
| P07 | 同一参数多处不一致 | 止损在技术面$87 vs 投资结论$90 | 全文搜索关键参数词 |
| P08 | 增速跳跃无解读 | 2025E +1.6% → 2026E +14.8%无解释 | 检查预期表后是否有解读段落 |
| P09 | 估值简化未标注 | DCF用简化两阶段但不说明 | 检查是否标注"简化"+"矩阵为权威参考" |
| P10 | 缺历史估值位置 | 只列当前PE不分析历史位置 | 检查Section 1.3 |
| P11 | ⚠️⚠️ 股价涨跌幅臆造 | 将"财报后暴跌"当作"1个月滚动涨跌幅" | 检查是否来自Finviz直接读取 |
| P12 | ⚠️ 保守中枢缺失 | 综合评级只给一个中枢 | 检查是否包含双中枢+当前价% |
| P13 | ⚠️ Bull Case论据空缺 | 仅列标题无数据支撑 | 检查每条是否附论据 |
| P14 | ⚠️ 跟踪指标与风险脱节 | 核心风险变量不在跟踪表中 | 对比风险top3 vs 跟踪表 |
| P15 | ⚠️⚠️ Non-IFRS口径混淆 | DCF中SBC用错口径 | 检查SBC/D&A是否与Reconciliation一致 |
| P16 | ⚠️⚠️ D&A反推而非附注 | D&A从EBITDA反推 | 检查来源是否标注"年报附注" |
| P17 | ⚠️⚠️ "估"标记过多 | 初稿30+处未经闭环补全 | 全文搜索"估""~"统计>5%则触发闭环 |
