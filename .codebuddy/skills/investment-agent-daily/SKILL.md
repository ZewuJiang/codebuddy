## RULE ZERO — 训练数据全面禁用令（最高优先级）

> **本报告呈报大老板的实时全球策略简报。每一个数据、每一个事件、每一个政策状态，都必须且只能来源于当期实时搜索结果（web_search / web_fetch）。严禁使用AI训练数据。**
>
> 核心三问自查：①这个信息是我刚才搜索到的吗？②搜索结果的时间戳在24小时内吗？③大老板追问能指向具体搜索结果吗？——任一为否，**该段重新搜索重写**。

## 报告定位

> **读者**: $10B级个人投资者，集中持仓（约12个标的），低频交易，时间极宝贵
> **风格**: 段永平/巴菲特式价值投资 + 全球宏观视野
> **核心宗旨**: 对过去24小时二级市场的策略信息和动态保持**敏锐捕捉**。框架是骨架，灵活应变是灵魂——不死板套模板，当日市场有什么重要的就写什么，模板服务于内容而非内容服务于模板。
> **目标**: 5分钟读完，回答三个核心问题：
> 1. **过去24小时发生了什么？**
> 2. **这对我的持仓有影响吗？**
> 3. **我需要采取行动吗？**
>
> **不是机构研究报告**，不需要VaR/蒙特卡洛/十维评分。
> 是大老板每天早上喝咖啡时的**投资参谋简报**。

## 五段式报告架构（v17.1核心）

```
§1 今日核心结论（结构化bullet point + 三大判断 + 行动建议）
  ↓
§2 全球市场隔夜速览（分区域精简表 + 含月度上下文）
  ↓
§3 重点标的与行业分析（核心个股表格+GICS板块+重点标的展开）
  ↓
§4 基金&大资金动向（10大机构 + 大师观点）
  ↓
§5 风险与机会雷达（红绿灯7-8项 + 阈值表 + 风险矩阵）
```

### §1 今日核心结论

> 用**结构化bullet point**概括过去24h最重要的事，最后给出**三大核心判断**（含置信度和逻辑链）和**行动建议**。
> 
> **呈现风格**：GS/MS/MBB投行简报风格，浅蓝底色卡片，左对齐，层次清晰。结论行字体加大突出，放在所有信息之后作为一锤定音。
>
> **⚠️ 灵活原则（最高优先级）**：模块不是死板模板，一切以当日实际市场动态为准。有什么写什么，没什么就不写——没有事件链就不写事件链，没有叠加因素就不写叠加因素，市场平淡时可能只需要2-3个模块。下面的模块仅作为**可选菜单**参考，不是必填清单。

**可选模块菜单**（根据当日实际情况灵活选用，非必填）：
- **事件链**：重大事件因果传导（有重大事件时用）
- **全球资产反应**：核心资产价格变动（有显著波动时用）
- **市场分化**：不同市场走势对比（有分化时用）
- **叠加因素**：其他影响因素（有时用）
- **正向/负向锚**：关键利好或利空（有时用）
- **📌 结论**：最终判断+操作建议（**必写**，永远放最后）

**格式示例**（重大事件日，模块较多）：
```markdown

>
> **事件链**：
```

### §3 重点标的与行业分析

> **核心原则：信息密度优先。核心个股表格+GICS板块表现每期必出，重点标的展开有事才写。**

**三层结构**：
1. **核心个股表格**（每期必出）：M7 + 重点AI标的 + 当日焦点个股，含收盘价/涨跌%/关键信号
2. **GICS 11板块表现**（每期必出）：全部11板块排名，含ETF/涨跌%/相对SPX/信号评级 + 轮动解读
3. **重点标的展开**（有事才写）：四栏卡片（事件/关键数据/为什么重要/投资含义）

**覆盖范围（雷达模式）**：
- **M7**: AAPL / MSFT / GOOGL / META / AMZN / NVDA / TSLA
- **AI产业链13环节**: 芯片设计→制造→光通信→存储(HBM)→服务器→网络→散热→电力→大模型→云计算→应用→终端AI→机器人
- **港股AI**: MiniMax / 商汤 / 百度 / 金山云 + IPO Pipeline（智谱/阶跃星辰/月之暗面/百川/零一/DeepSeek）
- **A股AI**: 寒武纪 / 中际旭创 / 新易盛等
- **金融代表**: GS / JPM / AXP（大行+消费信贷）
- **能源代表**: CVX / XOM（油气龙头）

**纳入标准（满足任一即报告）**：
- 重大订单（$10B+）或战略合作
- 财报超预期或大幅miss
- 管理层重大变动
- 产能/技术突破（CoWoS/HBM4/800G光模块等）

- 信贷爆雷/金融风险事件

**动态纳入原则**: 上述清单不固定。新IPO/新热点赛道龙头/突发事件涉及公司，只要影响投资逻辑，自动纳入。

## 工作流（4个阶段）

### 第零阶段：交易日检测 + 时间记录

```bash
date "+%A %Y-%m-%d %H:%M:%S"

```

**报告模板选择**：
- 周六 → 每周复盘模板（周五收盘情况 + 过去一周市场回顾）
- 周日 → 不出报告
- 周一 → 周一特殊模板（上周总结 + 本周展望 + 本周情景推演）

- 周二～周五 → 标准每日简报模板（不含回顾/展望/推演）

### 第一阶段：实时数据采集

**采集批次**：

| 批次 | 内容 | 搜索次数 |
|------|------|---------|
| 0 | 全球财经媒体头条扫描（Bloomberg/Reuters/WSJ + 华尔街见闻/第一财经/智通财经/格隆汇） | 2-3次 |

| 4 | 基金&大资金动向（13F/大师发言/机构调仓） | 1-2次 |

**周一额外批次**：
- 批次5: 上周市场周度数据
- 批次6: 本周关键事件日历

### 第二阶段：撰写报告

基于采集数据撰写五段式简报（§1-§5），控制在200-300行。

**撰写原则**：
- §1核心结论：基于批次0头条扫描，用结构化bullet point分模块呈现（事件链→资产反应→市场→结论），结论放最后
- §2市场速览：只报已收盘市场精确数据，未开盘标注"盘前"
- §3重点标的：核心个股表格+GICS板块每期必出，重点标的展开有事才写
- §4聪明钱：有动向才详写，无动向一句话带过
- §5风险雷达：红绿灯+日历+风险提示，精简有力

**简易终审（10项）**：
1. 所有数据来自当期搜索（RULE ZERO）

3. 涨跌幅方向正确
4. 无禁用词
5. 操作建议可执行
6. 行数200-300范围
7. 无代码块（全表格化）
8. 标题全中文
9. 全文数据一致
10. 文件名时间戳真实

2. **`投资Agent-每日策略简报-{YYYYMMDD}-{HHMM}.pdf`** — PDF（单页长图）

存放目录: `workflows/`

**文件命名规则**：
- `{HHMM}`取报告实际开始时刻系统时间
- 报告末尾免责声明区域"数据生成时间"与文件名时间戳一致
- 报告开头不放元数据块（生成时间/框架版本），保持标题直入正文
- 周六复盘命名: `投资Agent-每周复盘-{YYYYMMDD}-{HHMM}.md`

---

## PDF渲染铁律（精简版）

- CSS `font-family`: `"STHeiti", "Hiragino Sans GB", "Noto Sans CJK SC", "PingFang SC", sans-serif`
- 必须使用`md_to_pdf.py`两轮渲染法（Probe测高度→精确高度渲染）
- 严禁flag emoji（如🇨🇳🇺🇸），用文字替代
- 表格单元格≤25字，最多7列
