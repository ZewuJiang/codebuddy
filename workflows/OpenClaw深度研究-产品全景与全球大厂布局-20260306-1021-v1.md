## M1 行业概览及趋势洞察

### 1.1 AI Agent 行业定义与三大底层驱动力

**Ø 核心论点：AI Agent 正从「建议型工具」进化为「自主行动型劳动力」，2024-2026 年三大底层驱动力共振，推动行业穿越概念验证阶段进入规模化部署元年。**

**a. 驱动力一：模型能力跨越信任阈值**

推理能力的质变是 Agent 落地的第一块拼图。2024 年 OpenAI o1 系列引入强化学习 + 链式思维（Chain-of-Thought）推理，首次在复杂编码、数学与多步骤规划任务上展现出「可被信任」的决策质量——人类开始敢于交出系统权限。2025 年 Claude 3.5 Sonnet / Opus 在 SWE-bench Verified 上达到 49-72% 解题率；2026 年初 Claude Opus 4.5 进一步推至 **80.9%**，已超越多数中级工程师的基线表现。¹

> 1. 关键数据点：Gartner 预测到 **2028 年**，至少 **15%** 的日常工作决策将由代理型 AI 自主完成（2024 年该比例为 **0%**）。²

**b. 驱动力二：Scaling Law 边际递减 → 增长曲线从「博学」转向「能干」**

当单纯增加模型参数带来的能力增量开始放缓，行业投入的重心从预训练（pre-train bigger models）转向后训练 + 工具使用 + 环境交互。这直接催生了 Agent 架构的主流化：模型不再只做「一次性回答」，而是规划-执行-观察-调整的闭环循环。中信证券报告判断：「当前 AI 正由辅助工具向自主劳动力跨越，Agent 化转型驱动 Token 需求井喷。」³

**c. 驱动力三：商业模式从「卖 Token」转向「卖工时」**

SaaS 行业正在经历定价革命——从按 API 调用量计费，向「按任务完成付费（outcome-based pricing）」转型。Devin 按工程任务收费、Manus 以邀请制 + 订阅制切入、Claude Code 推出 $200/月 Max 计划，本质上都在验证同一假设：AI 直接替人完成劳动，商业价值天花板远高于工具辅助。Lumen Telecom 的案例已初步验证——单个 Agent 部署每年可节省 **$50M**，相当于 **187 名全职员工**的产出。⁴

### 1.4 技术趋势四大支柱

**Ø 核心论点：多模型路由、MCP/A2A 协议标准化、持久记忆系统、安全沙箱构成 AI Agent 技术栈的四根承重柱，缺一不可。**

**a. 多模型路由（Multi-Model Routing）**

- **问题**：单模型无法在所有任务上最优——推理用 o1/DeepSeek R1，编码用 Claude Opus，创意用 GPT-4o，成本敏感用 DeepSeek V3
- **解法**：OpenClaw 原生支持多 Provider 路由，用户可按任务类型自动切换模型
- **行业验证**：OpenRouter 2026 年 2 月数据显示中国 AI 模型调用量首超美国（**4.12 万亿 Token vs 2.94 万亿**），DeepSeek V3.2 / MiniMax M2.5 / Kimi K2.5 占据前五

**b. MCP/A2A 协议标准化**

- **MCP（Model Context Protocol）**：Anthropic 2024 年 Q4 提出，被比喻为「AI 的 USB-C 接口」——统一模型与外部工具/数据源的连接方式
- **A2A（Agent-to-Agent Protocol）**：Google 2025 年 Q2 推出，解决多 Agent 之间的协作通信
- **ANP（Agent Network Protocol）**：补充构成协议「黄金三角」
- **OpenClaw 原生集成**：通过 MCP 协议连接 40+ 工具（browser/canvas/cron/webhooks/skills），成为首批大规模验证 MCP 生态的消费者产品

**c. 持久记忆系统（Persistent Memory）**

- **问题**：传统 LLM 对话历史随 session 消失，Agent 无法「记住」用户偏好与长期上下文
- **业界方案**：Claude Code 的三层级记忆（工程级 Claude.md / 本地级 / 全局级）；OpenClaw 通过 Skills + STATE.yaml 实现跨会话状态保持
- **前沿方向**：Mem0（38.1k stars）等专用记忆管理框架正成为 Agent 技术栈标配

**d. 安全沙箱（Security Sandbox）**

- **问题**：Agent 具备真实执行能力（命令/文件/浏览器），权限过宽带来破坏性风险
- **OpenClaw 方案**：非主会话放入 Docker 沙箱 + VirusTotal 技能扫描 + 单用户信任边界 + 网关鉴权
- **行业共识**：Gartner 将 AI Agent 安全列为 2025-2026 年企业采纳的首要障碍；OpenClaw 的 `openclaw-ansible` 自动加固模板（UFW + Tailscale + Fail2ban + systemd hardening）被社区视为最佳实践

### 2.2 核心功能模块拆解

**Ø 核心论点：OpenClaw 的架构精髓在于「网关控制平面 + 多渠道接入 + 工具执行 + 技能生态」四层解耦设计，让 AI 从「聊天机器人」进化为「能做事的个人助手」。**

#### 模块一：入口层 — Channels（渠道即入口）

| 渠道 | 状态 | 说明 |
|:---|:---:|:---|
| WhatsApp | ✅ | 通过 Web API 桥接 |
| Telegram | ✅ | Bot API 原生集成 |
| Slack | ✅ | Slack App 模式 |
| Discord | ✅ | Bot 模式 |
| iMessage | ✅ | 通过 macOS Node 桥接 |
| 更多… | 🔄 | 社区持续贡献中 |

> **设计哲学**：不要求用户安装新 App，直接在日常使用的消息应用中召唤 AI 助手。这是 OpenClaw 区别于所有竞品的核心差异化——**渠道即入口，入口即壁垒**。

#### 模块二：控制平面 — Gateway（网关）

- **核心功能**：统一 WebSocket/HTTP 端口（默认 `127.0.0.1:18789`），管理会话路由、工具调度、事件分发、认证鉴权与设备配对
- **多代理路由**：支持将不同渠道/账户/peer 路由到隔离的 agent/workspace，实现一网关多场景
- **安全默认**：bind loopback + 强制 auth + WebSocket 源检查

#### 模块三：执行表面 — Tools（工具即能力）

| 工具类 | 能力 | 典型场景 |
|:---|:---|:---|
| **Browser** | 专用隔离浏览器 profile，可自动化网页操作 | 数据抓取、表单填写、网页监控 |
| **Canvas** | 可视化创作画布 | 生成图表、编辑图片 |
| **Cron** | 定时任务调度 | 每日新闻摘要、定期备份 |
| **Webhooks** | HTTP 回调触发 | 与 n8n/Zapier 编排集成 |
| **Shell/FS** | 命令执行 + 文件读写 | 代码运行、日志分析 |
| **Nodes** | 移动/桌面设备节点能力 | 相机拍摄、屏幕录制、GPS 定位 |

#### 模块四：持久记忆 — Memory & State

- **Skills + STATE.yaml 模式**：跨会话保持任务状态，支持自主项目管理
- **ClawHub 技能生态**：公共技能注册表，技能本质是包含 `SKILL.md` 的文件夹
- 支持搜索/安装/更新/发布与版本历史

#### 模块五：技能生态 — ClawHub

| 维度 | 现状 |
|:---|:---|
| **技能仓库 Stars** | ~4,100 |
| **技能格式** | 文件夹 + `SKILL.md`（声明式定义） |
| **安全机制** | VirusTotal 扫描 + 社区举报/隐藏机制 |
| **分发方式** | `openclaw install <skill>` CLI 命令 |
| **生态活跃度** | 日均新增 10-20 个社区技能（3月初数据） |

> **对标参照**：ClawHub 之于 OpenClaw ≈ npm 之于 Node.js ≈ App Store 之于 iPhone。技能生态的丰富度将决定 OpenClaw 的长期护城河深度。

#### 模块六：安全治理 — Trust & Sandbox

| 风险类 | 默认控制 | 加固建议 |
|:---|:---|:---|
| 提示注入 | 工具策略 + 权限分级 | 不受信内容用只读 profile；启用沙箱 |
| 网关暴露 | bind loopback + 强制 auth | 远程走 SSH/Tailscale，不裸暴露 |
| 工具权限过宽 | 默认 `tools.profile: messaging` | 关键动作审批 + 最小权限 |
| 供应链投毒 | VirusTotal 扫描 + 举报机制 | code review + 限制 skill 可用工具 |
| 节点滥用 | 设备配对与批准流 | 严格审批 + 最小能力暴露 |

> **加固最佳实践**：`openclaw/openclaw-ansible`（UFW + Tailscale + Fail2ban + 非 root 用户 + systemd hardening + Docker 隔离）被社区视为标准部署模板。

### 2.3 Skills 生态拆解

**Ø 核心论点：ClawHub 正在复刻移动互联网时代「App Store 飞轮」——低门槛发布、社区驱动筛选、安全扫描兜底，但生态成熟度仍处早期，距离「自运转飞轮」还需 2-3 个季度。**

**a. 技能架构**
- 每个 Skill 本质是一个文件夹，核心文件为 `SKILL.md`（声明技能能力、输入/输出、依赖）
- 支持链式组合（Chainable Skills）——多个技能串联完成复杂工作流
- 安装方式：`openclaw install <skill-name>` 或 ClawHub 在线安装

**b. 热门技能分类**

| 类别 | 代表技能 | 用途 |
|:---|:---|:---|
| **信息聚合** | reddit-digest, hn-summarizer, youtube-monitor | 每日资讯摘要推送 |
| **内容生产** | blog-writer, social-media-scheduler | 选题→写作→排版→分发 |
| **开发运维** | github-reviewer, ci-monitor, self-healer | 代码审查、CI 监控、自愈修复 |
| **生活助手** | family-calendar, grocery-tracker, health-log | 家庭日程、库存管理、健康追踪 |
| **数据分析** | csv-analyzer, web-scraper, market-watcher | 数据抓取与分析 |

**c. 生态成熟度评估**

### 2.5 核心优缺点

**Ø 核心论点：OpenClaw 的核心竞争力在于「渠道入口 × 本地控制 × 执行能力」的三角组合，但安全风险面与非多租户架构构成两大结构性短板。**

#### 竞争优势矩阵

| 优势 | 深度说明 | 竞品对比 |
|:---|:---|:---|
| **渠道即入口** | 无需切换 App，用已有 WhatsApp/Telegram/Slack/Discord 召唤助手 | Manus/Devin 需打开专用界面 |
| **本地优先** | 数据/密钥/部署位置/规则全在用户自有设备 | Claude Code 本地运行但模型推理上云；Cursor 完全云处理 |
| **行动能力** | Browser/Canvas/Cron/Nodes/Skills 让 AI 从建议变为执行 | ChatGPT 仍以对话为主 |
| **开源 MIT** | 完全开源，社区可审计、可定制 | Cursor/Devin 闭源；Claude Code 部分开源 |
| **架构可扩展** | 单一网关 + 多渠道 + 多代理路由 + 技能生态 | 大多竞品为单体架构 |

#### 结构性风险

| 风险 | 深度说明 | 严重度 |
|:---|:---|:---:|
| **提示注入** | 不受信网页/邮件/附件中的指令可诱导危险动作 | 🔴 高 |
| **权限过宽** | 执行命令、读写文件、浏览器自动化具有真实破坏性 | 🔴 高 |
| **渠道攻击面** | 攻击面延伸到 IM 平台（恶意私信/群聊/伪造事件） | 🟡 中高 |
| **非多租户** | 信任模型为「单用户/受信操作员边界」，不适合共享 | 🟡 中 |
| **数据非永远本地** | 模型推理仍可能通过第三方 API，消息平台也存储数据 | 🟡 中 |
| **供应链风险** | 技能市场与插件机制容易成为投毒入口 | 🟡 中 |

### 3.2 OpenAI：收编创始人的全盘棋

**战略意图**：通过招揽 OpenClaw 创始人 Peter Steinberger，OpenAI 试图将开源社区最强大的个人 Agent 创造者纳入体系，同时推进自有 Agent 基础设施（Responses API + Agents SDK）的标准化。

| 维度 | 数据 |
|:---|:---|
| **核心产品** | Codex App（50 万次下载）、Responses API + Agents SDK（开源） |
| **模型能力** | GPT-5.3-Codex 在 TerminalBench 2.0 达 77.3%（前代 64%）；SWE-bench Pro 57%；OSWorld 64% |
| **市场份额** | 62%（2024）→ 53%（2026E），仍为最大但份额持续收缩 |
| **关键动作** | 2025-03-11 发布 Agents SDK（开源）；Assistants API 计划 2026 年中停用 |
| **创始人招揽** | 2026-02-15 正式招揽 Peter Steinberger，信号明确——OpenAI 认为 Agent 是下一个 OS 级入口 |
| **财务压力** | 欠 NVIDIA/Microsoft/Oracle 超 **$1 万亿** 计算成本义务 |

**三步棋解读**：

a. **开源 Agents SDK**：通过开源框架层争夺开发者心智，对标 LangChain/CrewAI，但底层锁定 GPT 模型调用

b. **招揽创始人**：Peter Steinberger 不仅带来技术能力，更带来 253k stars 背后的开源社区影响力。OpenClaw 社区是否会因创始人转向而分裂，是 2026 年最大悬念之一

c. **Assistants API → Responses API 迁移**：从对话式 Agent 转向任务执行式 Agent，架构上与 OpenClaw 的 Gateway + 执行器模型趋同

> **风险评估**：OpenAI 的 $1 万亿计算成本义务是悬在头顶的达摩克利斯之剑。在 Agent 化转型的同时，商业化压力迫使其在开源（争夺开发者）与闭源（保护利润）之间反复摇摆。Agents SDK 开源但模型闭源的「半开放」策略能否持续，取决于 DeepSeek 等开源模型对其性能优势的侵蚀速度。

### 3.3 Meta：20 亿收购后的 Manus 进化

**战略意图**：Meta 以 $20-30 亿完成史上第三大收购，将 Manus 纳入体系，获得通用 Agent 赛道的先发优势和完整技术栈，同时与 Llama 开源生态形成闭环。

| 维度 | 数据 |
|:---|:---|
| **收购金额** | $20-30 亿（Meta 史上第三大收购） |
| **收购时间** | 2025-12-29/30，从初步接触到签署仅十余天 |
| **Manus ARR** | $1.25 亿（上线仅 8 个月） |
| **运营规模** | 处理 147 万亿 tokens，支撑 8000 万台虚拟计算机 |
| **人事安排** | 创始人肖弘出任 Meta VP |
| **模型协同** | Llama 4 Scout（10M tokens 上下文）+ MoE 架构 + 原生多模态 |

**收购逻辑拆解**：

a. **填补 Agent 短板**：Meta 在大模型（Llama）和社交平台（Facebook/Instagram/WhatsApp）上优势明显，但在 Agent 执行层几乎空白。Manus 的通用 Agent 能力恰好补齐这一短板

b. **速度优先**：十天内完成收购谈判，反映 Meta 对 Agent 赛道窗口期的极度紧迫感。OpenClaw 爆发后，「有能力的 Agent 团队」成为稀缺资产

c. **闭源策略风险**：Manus 被收购后从独立产品转为 Meta 内部工具，可能失去此前的社区基础和开发者生态。这与 OpenClaw 的开源路线形成鲜明对比

> **Ø 关键判断**：Meta 收购 Manus 验证了一条行业规律——**通用 Agent 创业公司的终局是被大厂收购**。Manus 8 个月 $1.25 亿 ARR 的数据虽亮眼，但独立发展面临的模型成本、算力投入和大厂竞争压力使得被收购成为理性选择。这对 OpenClaw 的启示是：**开源 + 社区 ≠ 免于被收编的护城河**，但 253k stars 的社区规模确实大幅抬高了「收编成本」。

### 3.5 Microsoft：Copilot Agent 化的生态壁垒

**战略意图**：依托 Microsoft 365 的 4 亿+ 月活用户基础和企业 IT 采购渠道，将 Copilot 从「对话助手」升级为「Agent 执行层」，在企业市场构建最深的 Agent 壁垒。

| 维度 | 数据 |
|:---|:---|
| **Copilot 付费席位** | 1500 万（2026 年初），同比增长 160% |
| **M365 月活** | 4 亿+ 用户 |
| **Agent 化产品** | M365 Copilot Sales Agent/Sales Chat、GitHub Copilot Agent Mode |
| **AI 投资** | 2025 财年资本开支 $800 亿+（主要用于数据中心和 AI 基础设施） |
| **GitHub Copilot** | Agent Mode Preview（多文件编辑 + 终端纠错），预告 Project Padawan（全自主编码） |

**壁垒解构**：

a. **企业渠道壁垒**：1500 万付费席位背后是全球最大的企业 IT 采购关系网。OpenClaw 在 C 端和开发者群体中势如破竹，但企业市场的 「预算审批 → IT 安全审计 → 合规验证」流程天然有利于 Microsoft

b. **数据壁垒**：M365 掌握企业邮件（Outlook）、文档（Word/Excel/PPT）、通讯（Teams）、代码（GitHub）的全链条数据，这是任何外部 Agent 产品无法触及的

c. **Agent Mode 进化缓慢**：GitHub Copilot Agent Mode 仍处于 Preview 阶段，Project Padawan 尚无明确时间表。相比 Claude Code 和 OpenClaw 的迭代速度，Microsoft 的 Agent 化转型显得保守

> **Ø 关键判断**：Microsoft 在 Agent 赛道的优势不在于产品创新速度，而在于**渠道垄断 + 数据锁定 + 预算惯性**。企业客户选择 Copilot 不是因为它最强，而是因为 「已经在用 M365」。这意味着 OpenClaw 在企业市场的渗透路径必须是**自下而上**（开发者先用 → 团队采纳 → 企业采购），而非自上而下。

### 3.6 Google：模型 + 平台双轨赋能

**战略意图**：以 Gemini 2.0 为底座，一次性发布四个 Agent 产品覆盖编程/浏览器/通用助手/游戏四大场景，同时推出 A2A（Agent-to-Agent）协议争夺多智能体协作标准。

| 维度 | 数据 |
|:---|:---|
| **Agent 产品矩阵** | Jules（编程）、Project Mariner（浏览器）、Project Astra（通用助手）、Gaming Agent |
| **底座模型** | Gemini 2.0（全面支撑 Agent 化转型） |
| **协议标准** | A2A 协议（Agent-to-Agent，多智能体协作） |
| **企业平台** | Agentspace 并入 Gemini Enterprise（预置 Agents + 无代码自建 + 企业数据连接） |
| **搜索整合** | AI Overviews 月活超 10 亿用户 |

**双轨策略**：

a. **模型层**：Gemini 2.0 的多模态能力是 Google 的核心差异化。Agent 需要「看」（视觉）、「听」（语音）、「做」（操作），Google 在多模态上的积累使其 Agent 产品天然具备更强的环境感知能力

b. **平台层**：Agentspace 并入 Gemini Enterprise 是典型的 Google 打法——用企业 SaaS 包裹 AI 能力，通过 Google Workspace 渠道分发。与 Microsoft Copilot 的竞争本质上是「Google Workspace vs M365」的延伸

c. **A2A 协议野心**：如果说 MCP 是「Agent 如何调用工具」的标准，A2A 则是「Agent 如何协调彼此」的标准。Google 试图在更高层（多智能体协作）建立标准话语权

> **风险评估**：Google 的 Agent 产品矩阵看似完整，但面临典型的「Google 陷阱」——产品多、聚焦少、杀不死又长不大。Jules/Mariner/Astra/Gaming Agent 四条线同时推进，内部资源竞争可能导致每条线都无法达到 Claude Code 或 OpenClaw 的单点突破深度。

### 3.7 SaaS 平台 Agent 化浪潮：Notion / Perplexity / Salesforce

#### Notion：从工具到 Agent 平台的范式转移

Notion 的 Agent 化进化路径是 SaaS 企业转型的最佳样本：

- **2025-11**：Notion 3.0 引入 AI Agent 基础能力
- **2026-01-20**：3.2 版发布 MCP 企业审计功能
- **2026-02-24**：3.3 版发布 Custom Agents（完全自主 + MCP 集成 Linear/Figma/HubSpot）
- **2026-05-04**：从订阅制切换为 **Notion Credits 计费**（从 token 到任务价值的范式转移）

> Notion Credits 计费模式的意义远超 Notion 本身——它标志着 SaaS 行业从「座席/订阅」向「按任务价值付费」的计费范式迁移。当 Agent 替代人类完成任务时，「人头/座席」的计费逻辑不再成立，「完成了什么任务」成为新的价值锚点。

#### Perplexity Computer：$200/月的多模型 Agent 终端

- **发布时间**：2026-02-27
- **定价**：$200/月（高端定位，面向专业用户）
- **核心能力**：19 种 AI 模型自动路由 + 多模型同时查询 + 子代理创建 + 完全云端执行
- **与 OpenClaw 对比**：Perplexity Computer 是闭源 + 云端 + 高价；OpenClaw 是开源 + 本地 + 免费。两者面向完全不同的用户群

#### Salesforce Agentforce：企业 Agent 的先行者

- **发布时间**：2024-09-12（行业最早的企业级 Agent 平台之一）
- **核心架构**：Atlas Reasoning Engine + Agent Builder + Partner Network
- **标志性案例**：Klarna 砍掉 Salesforce 后利润增加 $4000 万——Agent 正在颠覆传统 SaaS 的价值链

> **Ø 关键判断**：Klarna 案例是 Agent 时代对 SaaS 行业的「第一声枪响」。当 AI Agent 能直接完成客服、销售、运营任务时，「座席费」模式的 SaaS 面临被降维打击的风险。Salesforce 的 Agentforce 本质上是一场自我革命——用 Agent 能力替代自己赖以生存的 CRM 人工座席模式，否则就等着被 OpenClaw 这样的开源方案替代。

### 4.2 云厂商卖铲战深度分析

> **Ø 核心观察**：OpenClaw 爆发后，国内六大云厂商在 72 小时内全部上架一键部署服务，速度之快反映了两个信号——① 云厂商对 AI Agent 流量入口的极度渴望；② OpenClaw 的标准化架构（Docker + API）使得适配成本极低。本质上，这是一场**「卖铲子给挖金者」**的经典商业模式复现。

#### 六大云厂商 OpenClaw 布局对比矩阵

**卖铲战深层逻辑**：

a. **流量入口争夺**：OpenClaw 用户需要云服务器来部署 Agent，这直接转化为云厂商的新增 IaaS 收入。按保守估算，每个活跃 OpenClaw 实例月均消耗 ¥200-500 云资源，800 万国内月活意味着潜在 ¥16-40 亿/年的增量 IaaS 市场

b. **模型调用量争夺**：云厂商通过 OpenAI 兼容 API 接口将自有模型（或合作模型）接入 OpenClaw，争夺的本质是每一次 Agent 执行背后的 Token 消耗。这与 M1 章节讨论的 Token 经济直接相关

c. **差异化路径分化**：
  - 阿里云走「云电脑 + 全模型兼容」路线，强调桌面级体验
  - 腾讯云走「教程 + 生态」路线，以开发者教育建立心智
  - 百度智能云走「Skills 开放 + Coding Plan」路线，深度绑定 Agent 能力层
  - 火山引擎走「多端集成 + 安全」路线，四端方案覆盖面最广
  - 天翼云走「大额免费 Token」路线，以价格战抢占入门用户

> **Ø 关键判断**：国内云厂商的 OpenClaw 卖铲战本质上是 IaaS 价格战的延续——换了一个新流量入口，但竞争逻辑不变。真正的差异化在于**谁能从「部署平台」升级为「Agent 运营平台」**——提供 Skills 市场、数据治理、多端管理、安全审计等增值服务。百度智能云（七款 Skills）和火山引擎（四端+安全+记忆库）在这个方向上走得最远。

| **Bull** | 25% | GPT-6 实现 AGI 级 Agent 能力；OpenAI 整合 OpenClaw 创始人成功构建统一 Agent OS | OpenAI + Anthropic + Microsoft 三家合计占 Agent 市场 70%+；OpenClaw 社区萎缩至 <50k 活跃用户 |

#### 路线二：开源生态颠覆

| 情景 | 概率 | 假设条件 | 2028 年预期结果 |
|:---|:---:|:---|:---|
| **Bull** | 20% | DeepSeek/Qwen 系列持续缩小与 GPT 系列差距至 <5%；OpenClaw 完成商业化转型 | 开源 Agent 占全球 Agent 市场 40%+；OpenClaw 成为 Agent 时代的「Linux」 |

#### 路线三：SaaS 反击成功

| 情景 | 概率 | 假设条件 | 2028 年预期结果 |
|:---|:---:|:---|:---|
| **Bull** | 15% | Salesforce/ServiceNow/Notion 成功转型 Agent 平台；企业客户优先选择已有供应商 | SaaS+Agent 占企业 Agent 市场 50%+；纯 Agent 公司被边缘化 |

### 5.3 Token 经济全景 + 商业模式演变

#### Token 经济三阶段演变

| 阶段 | 时间 | 计费模式 | 代表案例 | 特征 |
|:---|:---|:---|:---|:---|
| **Phase 1：Token 计价** | 2023-2025 | 按 Token 数量计费 | GPT-4 $30/百万 tokens；DeepSeek $0.014/百万 tokens | 成本透明但难以反映任务价值 |
| **Phase 2：任务/积分计价** | 2025-2027 | 按任务完成度或积分计费 | Notion Credits（2026-05）；Perplexity Computer $200/月 | 从「消耗量」向「价值量」转移 |
| **Phase 3：结果计价** | 2027+ | 按业务成果计费 | Klarna 模式：省 $4000 万 → 按节省金额分成 | 与客户利益完全对齐，但需要可度量的 ROI |

**价格锚点变迁**：

DeepSeek 的极致低价（$0.014/百万 tokens）将 Token 价格推至成本线附近，迫使行业提前进入 Phase 2。当 Token 本身无法成为利润来源时，**差异化只能来自上层**——更聪明的路由（OpenClaw 多模型路由）、更好的任务完成率（Anthropic 75% 高端模型采用率）、更深的企业集成（Microsoft Copilot + M365）。

> **Ø 关键判断**：Token 经济的演变是理解整个 Agent 行业商业化路径的钥匙。Phase 1 利好模型厂商（收 Token 费），Phase 2 利好平台厂商（收任务费），Phase 3 利好行业解决方案商（收成果分成）。OpenClaw 作为开源平台，在 Phase 1-2 的商业化路径不清晰（用户自托管 = 平台无法收费），但在 Phase 3 有机会通过 Skills 市场抽佣实现变现。

### 5.5 三段式趋势预判

#### 短期（2026H2）：Agent 基础设施标准化

- MCP 协议升级至 v2.0+，成为行业事实标准；A2A 协议进入 Beta
- OpenClaw 月活突破 1500 万（全球），Skills 市场突破 2000 个
- 至少 2 家国内模型厂商完成港股 IPO（智谱/MiniMax 已完成）
- 企业 AI 支出突破 $12 亿年均水平，Agent 预算占比从 15% 升至 25%
- **Gartner 预测确认**：15% 日常工作决策将由 Agentic AI 自主完成（2028 目标的前奏）

#### 中期（2027）：商业化拐点

- Token 经济进入 Phase 2（任务/积分计价成为主流）
- OpenClaw 面临「商业化还是保持纯开源」的关键抉择——Foundation 模式 vs 商业公司模式
- 至少 1 起 $10B+ 的 Agent 领域并购（可能目标：Perplexity、Notion 或 Dify）
- AI 生成代码比例从 41% 升至 55%+
- 企业 Agent 采用率从当前的 30% 升至 50%+，但**信任鸿沟**（仅 29-46% 信任 AI 输出）仍是主要障碍

#### 长期（2028+）：Agent 操作系统化

- Agent 成为操作系统级入口，类似智能手机时代的 iOS/Android 格局形成
- 全球 Agent 市场规模达 $47B+（2030 基准预测）
- 胜出者将在「平台巨头」和「开源生态」之间产生——两极格局定型
- 编码自动化比例突破 60%，软件工程师角色从「写代码」向「指导 Agent 写代码」转变
- Token 经济进入 Phase 3（结果计价），SaaS 行业完成 Agent 化转型或被淘汰
