## 0. 你最关心的结论（先给老板看的 1 页）

- **行业主线已经从“更聪明的对话”转向“可控的行动力”**：头部厂商集中把能力落在三件事上：
  - **统一的 Agent 开发接口**（例如 OpenAI `Responses API` + `Agents SDK`），把“多步任务+工具+可观测”收敛成稳定的工程面。
  - **Computer-use / Desktop automation**（例如 OpenAI 内置 `computer use` 工具、Anthropic `computer use tool`），用“截图+鼠标键盘事件”扩展到任何 UI 系统。
  - **企业级治理与权限**（Notion 对 `MCP` 的企业审计/连接控制、Databricks 绑定 `Unity Catalog`、Salesforce 的 `Atlas Reasoning Engine` + 低代码构建）。
- **“协议化/插件化”成为分发生态的关键**：Notion 在 2026-01-20 明确把 `MCP` 的审计与连接控制纳入企业能力，并在 3.3 的 Custom Agents 里把 MCP 当作外部工具生态的标准接入方式（Linear/Figma/HubSpot 等）。
- **开发者侧进入“可观测+护栏”标配阶段**：OpenAI 在 `Agents SDK` 中把 tracing/guardrails/handoffs/sessions/human-in-the-loop 做成一套生产级原语；GitHub Copilot Agent Mode 把“多文件编辑 + 终端迭代纠错”产品化。
- **商业化开始从 token 叙事迁移到“按任务/按积分/按流程价值”**：Notion 3.3 明确 2026-05-04 开始按 `Notion credits` 计费；微软在 Copilot 中公开强调“用 agents 改造每个业务流程”；Snowflake 的 2026 预测把 Agents 视作主导形态。

> 结论一句话：**OpenClaw 的竞争窗口在“把 Computer-use + MCP 生态 + 企业治理/可观测”做成一体化工程平台**，而不是再做一个“更会聊天的壳”。

## 3. 横向趋势图谱（把“动作”抽象成 6 条主线）

1) **统一化的 Agent 接口与工程栈**
- OpenAI：`Responses API` 将 Chat Completions 的简洁与 Assistants 的工具能力合并；并给出 Assistants API 计划在 2026 年中停用（迁移到 Responses）。

2) **Computer-use（UI 级通用执行）进入主战场**
- OpenAI：内置 `computer use` 工具，能力来自 Operator 同款 CUA 模型。
- Anthropic：`computer use tool` 以标准化工具协议输出（截图/坐标/键鼠事件），并强调隔离环境、提示注入与人类确认。

3) **协议化生态（MCP 等）成为“跨应用工作流”的低摩擦形态**
- Notion：在移动端版本中明确 MCP 审计与“连接控制（即将推出）”，Custom Agents 通过 MCP 连接 Linear/Figma/HubSpot 等。

4) **企业治理：权限、审计、数据边界与可观测成为标配**
- Databricks：Agent 绑定 Unity Catalog（治理/血缘/权限），并在执行代码前要求用户批准。
- OpenAI Agents SDK：Tracing/Guardrails/Sessions/Human-in-the-loop。

5) **分发入口从“聊天框”扩展到“IDE/协作/CRM/办公套件”**
- GitHub：Agent Mode 与 Copilot Edits 面板融合，多文件+终端迭代。
- Microsoft 365：agents 直接嵌入销售工作流，并连接 Dynamics 365 / Salesforce。
- Salesforce：Agentforce 作为“数字劳动力平台”，通过 AppExchange/Partner Network 分发。

6) **计费从 token 迁移到任务价值（credits / workflow ROI）**
- Notion：2026-05-04 起 Custom Agents 按 Notion credits 计费。
- Microsoft：强调“用 agents 改造每个业务流程”，并披露 Copilot Studio 自定义 agents 的规模数据。

## 5. 对 OpenClaw 的可执行启示（落到产品/架构/商业化）

### 5.1 产品：三件套优先级
- **P0｜Computer-use 执行层**：把“截图→定位→动作→反馈”的循环做成可靠工程能力（参考 Anthropic 的风险防护清单：隔离环境、域名白名单、提示注入、人工确认）。
- **P0｜MCP 生态层**：把 MCP 当作“外部工具/应用”的标准接口，并补齐企业能力：连接控制、审计日志、权限继承（Notion 已在企业侧公开推进）。
- **P1｜可观测与护栏**：Tracing（链路）、Guardrails（输入输出与工具调用策略）、回放与评测（对齐/稳定性）。OpenAI Agents SDK 的原语可以作为“对标清单”。

### 5.2 工程：建议的参考架构（从最小可用到企业版）
- **最小可用（MVP）**：单 Agent + 工具（MCP/HTTP）+ 任务状态机 + 基础日志。
- **可扩展（Pro）**：多 Agent 编排（handoff/manager）、统一工具网关、细粒度权限、可回放 tracing。
- **企业版（Enterprise）**：审计与合规（SOX/ISO27001）、数据域隔离、连接器白名单、成本与配额、组织级策略。

### 5.3 商业化：从“卖 token”迁移到“卖任务/卖工时”
- 参考 Notion credits：
  - **计费单元**：按任务复杂度/动作次数/外部工具调用次数（可含 computer-use 步数）。
  - **企业购买方式**：管理员统一购买 credits + 配额与审计。
