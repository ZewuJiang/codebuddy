### 2.1 左栏：多产品矩阵布局（修改版）

#### ① CoPaw —— 国产友好型个人 Agent

**标注：2/14 发布 → 2/28 开源 ｜ 通义 AgentScope 团队 ｜ Apache 2.0**

> CoPaw：自研类 OpenClaw 竞品，**国内 IM 生态适配最全**的个人智能体

**核心差异化（4 项关键优势）：**
- 🌐 **IM 全域接入**：原生打通钉钉、飞书、QQ、Discord、iMessage（OpenClaw 仅支持海外平台）
- ☁️ **端云一体部署**：本地 Python 环境 / 魔搭创空间 / 阿里云轻量服务器三种方式
- 🧠 **多模型灵活接入**：通义千问、OpenAI、DeepSeek + Ollama 本地模型均可
- 🐍 **Python 技术栈**：相较 OpenClaw 的 Node.js 22+，**大幅降低国内开发者门槛**

> **MBB 战略解读**：CoPaw 的核心策略是 **"国产化替代"**——精准击中 OpenClaw 对国内 IM 零适配的最大痛点，以 AgentScope 框架为底座预留企业级扩展空间。

---

#### ② HiClaw —— 开源多 Agent 团队协作系统（⚠️ 名称修正：非 HiPaw）

**标注：3/6 开源 ｜ 阿里云 Higress 团队 ｜ Apache 2.0 ｜ GitHub: higress-group/HiClaw**

> HiClaw：行业唯一的 **Team 版 OpenClaw**，从"单兵作战"→"AI 军团协作"

**核心架构——Manager / Worker 双角色体系：**

```
用户需求 → Manager Agent（AI管家）
              · 需求理解与任务拆解
              · 智能分配给 Worker
              · 进度追踪与结果汇报
            ↓
    ┌───────┼───────┐
    Worker 1    Worker 2    Worker 3
    前端         后端         测试
  (独立Docker) (独立Docker) (独立Docker)
```

**HiClaw vs OpenClaw 关键差异：**

| 痛点 | OpenClaw | HiClaw 方案 |
|------|----------|------------|
| **API Key** | 散落配置，安全风险 | **统一密钥管理** |
| **Agent 协作** | 单兵作战 | **Manager 自动拆解 + 分派 + 追踪** |
| **容器隔离** | 同一环境 | **每个 Worker 独立 Docker 容器** |
| **目标用户** | 个人用户 | **面向团队 / 企业级场景** |

> **MBB 战略解读**：HiClaw 是阿里在 OpenClaw 赛道上**独有的差异化产品**——目前所有竞品（字节 ArkClaw、腾讯 WorkBuddy、智谱 AutoClaw 等）均无类似的多智能体团队协作系统。Higress 团队在 API 网关和安全隔离方面的深厚积累，为 HiClaw 的企业级安全性提供了技术背书。

---

#### ③ QoderWork —— 桌面级 AI 智能助理

**标注：1/30 Mac版 → 3/3 Windows版 ｜ Qoder 团队**

> QoderWork：桌面端 AI 智能助理，**对标 MiniMax Cowork**，从编程扩展到全场景办公

**核心能力：**
- 💻 **双平台覆盖**：macOS + Windows
- 🔒 **本地安全运行**：文件不上云，仅指令加密上传
- 🛠️ **MCP 工具集成**：调用百炼大模型 API + Skills 市场扩展
- 🎯 **Quest 模式**：多 Agent 协作编码，描述需求→自动生成代码
- 📊 **3/3 新增**：模型分级选择器，大幅降低使用成本
