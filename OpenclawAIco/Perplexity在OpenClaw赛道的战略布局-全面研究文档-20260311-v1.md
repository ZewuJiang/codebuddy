## 2. ⭐ Perplexity Computer——自研OpenClaw

> **这是Perplexity在OpenClaw赛道最核心、最重量级的布局——对标Anthropic OpenClaw，但CEO定义为"更安全的OpenClaw"和"真正的通用数字工作者"。**

### 2.1 发布信息

| 维度 | 详情 |
|------|------|
| **产品名称** | Perplexity Computer |
| **发布日期** | 2026年2月25-27日（本周上线） |
| **可用范围** | 仅限 **Perplexity Max** 订阅用户（$200/月） |
| **积分配额** | Max用户每月10,000积分 |
| **CEO定位** | "2026年个人电脑应有的样子" / "更安全的OpenClaw" |
| **研发周期** | 团队秘密研发两个月 |

### 2.2 核心能力

Perplexity Computer 被官方定义为 **"an agentic AI system"**（智能体AI系统），官方称其 **"unifies every current AI capability into a single system"**（将所有当前AI能力统一到一个系统中）。

**七大核心能力矩阵**：

| 能力 | 说明 |
|------|------|
| **推理（Reason）** | 多步骤逻辑推理，处理复杂决策 |
| **委派（Delegate）** | 自动创建子代理（sub-agents）处理特定子任务 |
| **搜索（Search）** | 基于Perplexity原生搜索能力的实时信息获取 |
| **构建（Build）** | 创建网站、可视化图表、仪表盘等数字产品 |
| **记忆（Remember）** | 上下文记忆管理，跨任务信息保持 |
| **编码（Code）** | 代码生成、调试、部署能力 |
| **交付（Deliver）** | 完成项目部署、文件生成等最终交付物 |

### 2.3 技术架构：元路由器（Meta-Router）+ 19个模型协同

Perplexity Computer 的核心竞争力在于其 **多模型编排架构**，而非依赖单一模型：

```
                    ┌────────────────────────────┐
                    │      元路由器 (Meta-Router)  │
                    │  ——任务拆解 + 模型路由——      │
                    └────────────┬───────────────┘
                                 │
          ┌──────────┬───────────┼───────────┬──────────┐
          ▼          ▼           ▼           ▼          ▼
    ┌──────────┐┌──────────┐┌──────────┐┌──────────┐┌──────────┐
    │Claude    ││Gemini    ││GPT-5.2   ││Grok      ││Veo 3.1   │
    │Opus 4.6 ││Flash     ││          ││          ││          │
    │核心大脑  ││视觉输出  ││超长上下文 ││低延迟    ││视频生成  │
    │推理+编码 ││          ││记忆      ││          ││          │
    └──────────┘└──────────┘└──────────┘└──────────┘└──────────┘
              + 其他14个专业化模型 = 共19个前沿模型
```

**模型分工（2025年12月使用数据）**：

| 任务类型 | 首选模型 | 说明 |
|----------|----------|------|
| 缜密逻辑+编程 | **Claude Opus 4.6** | 核心大脑，主导推理和编码 |
| 深度信息挖掘 | **Google Gemini** | 搜索和信息检索优势 |
| 视觉输出 | **Gemini Flash** | 图像理解和视觉生成 |
| 软件工程 | **Claude Sonnet 4.5** | 代码生成专用 |
| 医学研究 | **GPT-5.1** | 知识深度优势 |
| 低延迟任务 | **xAI Grok** | 快速响应场景 |
| 超长上下文记忆 | **GPT-5.2** | 大窗口上下文 |
| 视频生成 | **Veo 3.1** | 视频内容创作 |

**关键机制**：
- **自动路由**：系统根据任务类型自动选择最佳模型，用户无需手动指定
- **子代理创建**：可自动创建多个子代理并行处理不同子任务
- **异步多任务**：用户可在后台同时运行数十台 Perplexity Computer
- **长期任务**：支持持续运行数小时甚至数月，仅在"真正需要真人介入"时通知用户
- **成本优化**：使用修改过的开源中国构建的LLM来更便宜地回答常规查询
- **Model Council**：允许用户同时查询多个模型，获取多角度答案

### 2.4 运行环境与安全性

**与 OpenClaw 的关键区别——安全沙盒机制**：

| 维度 | Anthropic OpenClaw | Perplexity Computer |
|------|-------------------|-------------------|
| **运行环境** | 直接访问操作系统，可操作本地文件和应用 | 限制在浏览器**安全沙盒**内，不触碰主系统 |
| **安全事件** | 曾发生删除Meta高级总监所有邮件的事故 | 沙盒隔离，即使出现漏洞也不扩散到主网络 |
| **权限模型** | 需用户授权系统级操作 | 高风险操作需明确用户批准；API密钥仅作用于当前会话 |
| **本地vs云端** | 本地+云端混合 | **完全云端运行** |
| **CEO评价** | - | "更安全的OpenClaw" |

**安全设计要点**：
- 高风险操作（金融交易、发送通信、修改外部系统）**必须**获得用户明确批准
- API密钥和登录会话仅作用于当前工作流会话，**不会持久化保留**
- 按任务临时授权，而非全局授权

**已知局限**：
- 复杂多步工作流可能遇到任务图结构脆弱、子任务失败级联的问题
- 每个链式步骤都会增加幻觉风险
- 高频自动化调用可能被Google等巨头视为"恶意使用"而封禁访问

### 2.5 应用场景与标杆案例

**"平民版彭博终端机"事件**：
- 发布后仅 **5小时**，就有网友用 Perplexity Computer 成功复刻了一个"平民版彭博终端机"
- **原版彭博终端机**：金融从业者标配，2025-2026年单用户年费约 **$25,000-$30,000**（约合¥14.5万-19.3万）
- **Perplexity Computer**：仅需 Max 订阅 **$2,400/年**，仅为彭博年费的 **1/12**

**其他典型应用场景**：
- 收集统计数据、金融或法律数据
- 创建分析报告
- 将发现结果作为网站或可视化图表分享
- 项目部署、文件设计
- 课题研究、工作流管理

### 2.6 发布花絮

> 原定在媒体简报会上演示 Perplexity Computer，但**在活动开始前几小时因发现产品缺陷而取消演示**。这一细节反映了该产品仍处于早期阶段的现实。

## 3. Comet 浏览器——AI原生入口重构

### 3.1 核心信息

| 维度 | 详情 |
|------|------|
| **产品名称** | Comet（彗星浏览器） |
| **桌面版发布** | 2025年7月9日 |
| **Android版发布** | 2025年11月20日 |
| **iOS版** | 2026年下月登陆（截至2026年2月） |
| **可用范围** | 首先向 Max 订阅者开放，后扩展至 Pro |
| **定位** | AI驱动的Web浏览器，旨在取代Chrome/Safari |

### 3.2 核心功能

| 功能 | 说明 |
|------|------|
| **AI原生搜索** | 可将Perplexity设为默认搜索引擎 |
| **标签页智能交互** | 可提及特定标签页向助手提问；语音模式跨标签页提问 |
| **跨标签页总结** | 助手可总结所有打开标签页的搜索内容 |
| **代理研究与购物** | 浏览器可代表用户进行研究或购物，用户可查看Agent执行动作 |
| **内置广告拦截** | Android版内置广告拦截器 |
| **复杂任务处理** | 可执行长时间运行的复杂任务（如将网站数据传输到电子表格） |

### 3.3 即将推出的功能

- **对话代理**：跨网站搜索并执行操作的对话式Agent
- **快捷方式**：为助手创建快捷方式执行快速操作
- **密码管理器**：功能齐全的密码管理器

### 3.4 战略意义

Comet 代表了 Perplexity 从"搜索应用"到"浏览器入口"的**流量控制权升级**：
- **Android优先**：因为多个运营商和OEM厂商请求预装Comet（此前已与摩托罗拉合作预装Perplexity App）
- **竞争格局**：OpenAI、Opera、The Browser Company（Dia）等均已推出AI浏览器，但大多集中在桌面端
- **安全考量**：Perplexity 2025年10月博文承认AI代理可能带来安全风险，需从零开始重新思考安全架构

## 4. Perplexity Assistant——移动端AI Agent

### 4.1 核心信息

| 维度 | 详情 |
|------|------|
| **发布时间** | 2025年1月24日 |
| **平台** | Android（首发） |
| **定位** | 多模态AI助手，不只是问答而是执行任务 |
| **核心定义** | "多模式人工智能代理" |

### 4.2 核心能力

**多模态交互三要素**：
1. **语音交互**：语音提问和语音指令
2. **摄像头视觉**：直接启动相机，AI可以"看到"现实世界
3. **屏幕内容理解**：基于当前屏幕内容进行提问和操作

**跨应用任务执行（Multi-App Actions）**：

| 场景 | 执行流程 |
|------|---------|
| "帮我叫车" | 主动询问目的地 → 自动启动Uber → 显示可用车辆 |
| "写封邮件" | 理解意图 → 打开邮件应用 → 草拟内容 |
| "设定提醒" | 解析时间 → 创建日历事件/提醒 |
| "搜索音乐" | 打开音乐应用 → 搜索并播放 |
| "预订晚餐" | 查找餐厅 → 打开预订应用 |

### 4.3 与OpenAI Operator的对标

Perplexity Assistant 紧随 **OpenAI Operator** 之后发布，是Perplexity在移动端Agent领域的直接回应。区别在于Perplexity Assistant更侧重**手机原生操作**（叫车、打电话、控制应用），而非网页端的代理操作。

## 5. Deep Research——深度研究能力

### 5.1 核心信息

| 维度 | 详情 |
|------|------|
| **发布时间** | 2025年2月15日 |
| **发布背景** | 紧随Google Gemini（2024.12）和OpenAI Deep Research（2025.2）之后 |
| **可用范围** | **免费增值模式**：免费用户每天5次，Pro用户每天500次 |
| **命名** | Deep Research（与Google和OpenAI同名） |

### 5.2 核心功能

- **工作原理**：执行数十次搜索 → 阅读数百个来源 → 推理下一步 → 不断优化研究计划 → 生成全面报告
- **执行速度**：大多数任务在 **3分钟内** 完成
- **基准表现**：SimpleQA 基准测试准确率 **93.9%**，超越所有已发布模型
- **Draco基准**：Perplexity自研的复杂研究任务基准，Deep Research击败Gemini等竞品
- **输出格式**：详细报告，可导出为PDF或分享为Perplexity页面

### 5.3 技术特性

- 配备搜索和编码能力（可运行代码分析数据）
- 反复搜索、阅读文档，根据新信息动态调整研究计划
- "类似于人类专家研究新主题的方式"
- 能够处理金融、市场营销、技术等多个专业领域

### 5.4 竞品差异

| 维度 | Perplexity Deep Research | OpenAI Deep Research | Google Gemini Deep Research |
|------|------------------------|---------------------|---------------------------|
| **定价** | 免费增值（免费5次/天） | Pro专属（$200/月） | Ultra专属（$249.99/月） |
| **速度** | ~3分钟 | ~5-10分钟 | ~3-5分钟 |
| **免费使用** | ✅ 每天5次 | ❌ | ❌ |
| **代码执行** | ✅ | ✅ | ✅ |

## 7. Sonar API——B端搜索基础设施

### 7.1 核心信息

| 维度 | 详情 |
|------|------|
| **发布时间** | 2025年1月21日 |
| **定位** | 让企业和开发者将Perplexity搜索能力集成到自有应用 |
| **核心优势** | 实时联网+可信来源+引用+可定制来源 |
| **标杆客户** | **Zoom**（AI视频会议助手） |

### 7.2 版本与定价

| 版本 | 搜索费 | 输入费 | 输出费 | 特点 |
|------|--------|--------|--------|------|
| **Sonar（基础版）** | $5/1,000次 | $1/~100万tokens | $1/~100万tokens | 轻量快速 |
| **Sonar Pro（专业版）** | $5/1,000次 | $3/~100万tokens | $15/~100万tokens | 多次搜索，更详细 |

### 7.3 Sonar模型表现

- **Sonar-Reasoning-Pro-High**：在 LM Arena Search Arena 评估中**与Google Gemini-2.5-Pro-Grounding并列第一**（2025年4月14日发布）
- 基于 **Llama 3.3 70B** 进一步训练，运行在 Cerebras 推理基础设施上
- Token解码速度达 **1,200 tokens/秒**，是 Gemini 2.0 Flash 的 **8.5倍以上**

### 7.4 开发者生态

- **搜索索引自建**：不再依赖其他公司的网络索引API，拥有自己的AI优化搜索索引
- **"Ask"开发者大会**：计划于 **2026年3月11日** 在旧金山举行，推广Sonar API的第三方使用

## 9. 商业模式变革：从广告到纯订阅

### 9.1 广告模式的兴与废

这是Perplexity商业史上最重要的战略转折之一：

| 时间 | 事件 |
|------|------|
| 2024年下半年 | Perplexity成为**最早尝试在AI搜索中插入广告**的公司之一 |
| 2024-2025年 | 与超过 **300家出版商** 建立合作关系，广告收入分享 |
| **2026年2月** | 🔥 **完全放弃广告模式**，震惊行业 |

### 9.2 放弃广告的核心原因

> "广告破坏了用户对答案准确性的信任。"
> —— Perplexity 高管

具体逻辑链：
1. Perplexity的核心价值主张是 **"准确的答案"**
2. 广告的本质是 **推广特定内容**，与"客观准确"存在根本矛盾
3. 用户一旦怀疑答案受广告影响，**信任基础崩塌**
4. 因此宁可放弃短期收入，也要维护产品信任

### 9.3 转向纯订阅+API的收入结构

```
收入来源构成（2026年后）：
├── 消费者订阅
│   ├── Pro: $20/月（核心搜索+Deep Research+Labs）
│   └── Max: $200/月（所有功能+Perplexity Computer）
├── 企业订阅
│   ├── Enterprise Pro: $40/人/月
│   └── Enterprise Max: 即将推出
└── API 收入
    └── Sonar API: 按搜索次数+Token计费
```

### 9.4 用户增长策略的根本转变

高管明确表态：

> "你永远不会听到我们谈论MAU（月活跃用户），因为我们实际上并没有追求尽可能多的用户。"
> "现在早上查看的是最新的收入指标，而不是前一天的查询数量。"

**从"流量思维"到"价值思维"**：目标不是服务4,500万用户，而是服务那些做出**"影响GDP决策"**的人群。

## 15. 战略研判与核心洞察

### 15.1 Perplexity在OpenClaw赛道的战略定位

> **"不做操作系统级Agent，做浏览器沙盒级Agent。不做单模型Agent，做19个模型的最佳编排者。不做最多人用的Agent，做影响GDP决策的人的Agent。"**

### 15.2 五大核心战略洞察

#### 洞察一：搜索引擎基因是Agent竞赛的隐藏王牌

Perplexity做Agent的最大优势不是模型能力（他们没有自己的前沿大模型），而是**搜索基础设施**。在Agent需要"了解世界"的时候，Perplexity的实时搜索能力是原生的、快速的、有引用的——这是OpenAI和Anthropic做Agent时需要额外构建的能力。

#### 洞察二：多模型编排是差异化壁垒

当所有竞争对手都在用自家单一模型构建Agent时（Anthropic用Claude，OpenAI用GPT，Google用Gemini），Perplexity选择了**模型中立**的多模型路由策略。这意味着：
- 不会被任何单一模型的短板拖累
- 可以永远使用市场上最新最好的模型组合
- 模型越商品化，Perplexity的编排层价值越高

#### 洞察三：从广告到纯订阅的壮士断腕

放弃广告模式是一个极其大胆的决定——这等于放弃了潜在的数十亿美元规模的广告收入天花板。但这个决定的底层逻辑是清晰的：**在AI搜索中，信任比流量更值钱**。如果用户怀疑"这个答案是因为广告主付了钱"，Perplexity的核心价值主张就崩塌了。

#### 洞察四：从大众到精英的用户定位转型

$200/月的Max计划、"影响GDP决策的人群"的定位、放弃追求MAU的表态——这些信号都指向Perplexity正在从大众搜索引擎**转向精英知识工作者的生产力平台**。这与Bloomberg Terminal的商业模式有异曲同工之妙。

#### 洞察五：产品矩阵的"俄罗斯套娃"战略

Perplexity的产品布局呈现出清晰的层层嵌套结构：
```
搜索（基础层）
 └─ Deep Research（增强层）
     └─ Labs（生产力层）
         └─ Perplexity Computer（Agent层）
             └─ Comet 浏览器（入口层）
```
每一层都建立在前一层之上，用户越深入使用，越难以迁移。这是一个精心设计的**"用户锁定"金字塔**。

### 15.3 风险与挑战

| 风险 | 严重程度 | 说明 |
|------|---------|------|
| **模型供应链风险** | 🔴 高 | 依赖Anthropic/OpenAI/Google的模型，如果任何一方限制访问或大幅涨价，将直接影响服务 |
| **财务可持续性** | 🔴 高 | 仍在亏损，且多模型架构的推理成本极高 |
| **版权诉讼** | 🟡 中 | 多起诉讼未决，可能导致内容获取受限 |
| **巨头竞争** | 🟡 中 | Google/Microsoft/OpenAI的资源优势压倒性 |
| **安全风险** | 🟡 中 | 高频自动化调用可能被平台封禁 |
| **用户信任** | 🟢 低（已控制） | 放弃广告模式后信任风险降低 |

### 15.4 核心结论

Perplexity 在 OpenClaw 赛道的布局可以用一句话总结：

> **"用搜索做根基，用多模型做引擎，用浏览器沙盒做安全边界，用订阅做商业模式——打造一个'更安全、更聪明、更便宜'的通用数字工作者，服务于做出关键决策的精英知识工作者。"**

这是一个与Anthropic（深度系统控制）、OpenAI（通用智能平台）、Google（全栈生态）完全不同的切入角度——**轻资产、多模型、搜索原生、安全优先**。
