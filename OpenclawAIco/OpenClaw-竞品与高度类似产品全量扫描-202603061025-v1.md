# OpenClaw 竞品与高度类似产品全量扫描

> **版本**：v1 · 2026-03-06 10:25  
> **范围**：严格围绕与 OpenClaw **产品形态高度类似**的产品——即「本地/云端可自托管的个人 AI Agent 平台」，具备多渠道 IM 接入、工具执行、Skills/插件生态、持久记忆等核心特征。不涉及泛 Agent API（如 OpenAI Responses API）、纯编码 Agent（如 GitHub Copilot Agent Mode）或企业级 Agent 平台（如 Salesforce Agentforce）。  
> **证据原则**：官方来源优先标 🟢，权威媒体报道标 🟡，社区/二手信息标 🟠。无法核验的不写入。

## 二、竞品分类框架

根据与 OpenClaw 的产品形态相似度，我将竞品分为四个圈层：

```
┌─────────────────────────────────────────────────────┐
│          🔴 第一圈层：直接同类竞品                      │
│  产品形态几乎一致：本地/自托管 + 多渠道IM + 工具执行     │
│  + Skills生态 + 持久记忆                               │
│  → CoPaw / KimiClaw / MaxClaw / LobsterAI /           │
│    OpenClaw-CN                                         │
├─────────────────────────────────────────────────────┤
│          🟠 第二圈层：功能高度重叠的竞品                  │
│  具备「工具执行 + 自主任务完成 + 多模型」但运行方式不同    │
│  → Manus / Perplexity Computer / 扣子空间(Coze Space)  │
│    / Claude Cowork / OpenAI Operator                   │
├─────────────────────────────────────────────────────┤
│          🟡 第三圈层：部分功能重叠的产品                  │
│  在某些维度（computer use/本地运行/Agent框架）与           │
│  OpenClaw有交集                                        │
│  → Open Interpreter / Dify / AutoGPT / Amazon Nova Act │
│    / Google Project Mariner / 智谱 AutoGLM              │
├─────────────────────────────────────────────────────┤
│          ⚪ 第四圈层：广义个人AI助手（形态差异大）         │
│  → Apple Intelligence/Siri / Rabbit R1 / Humane AI Pin │
│    / Devin (纯编码) / Claude Code (纯CLI编码)           │
└─────────────────────────────────────────────────────┘
```

## 八、关键洞察与趋势判断

### 8.1 OpenClaw 的护城河

1. **开源 + 本地优先 = 数据主权**：在 AI Agent 需要控制用户电脑的场景下，"数据不离开设备"是核心诉求。OpenClaw 是目前唯一满足此条件且有规模生态的产品。
2. **IM 多渠道接入最全**：50+ 渠道接入能力是真正的"基础设施级"壁垒，竞品普遍在 3-5 个渠道。
3. **Skills 生态飞轮**：5,700+ 社区技能形成了"写 Skill → 用 Skill → 反馈迭代"的正循环，竞品难以短期追赶。
4. **GitHub 263k stars 的社区势能**：60 天登顶 GitHub 历史星标榜，社区贡献者构成持续迭代引擎。

### 8.2 OpenClaw 的短板

1. **安全漏洞频发**：CVE-2026-25253 + Shodan 上大量 18789 端口零认证暴露 → 企业级部署受阻。
2. **技术门槛高**：CLI 安装 + Node.js 22 + 配置复杂 → CoPaw/LobsterAI 等"极简安装"产品直接切此痛点。
3. **合规性差**：企业环境下的权限/审计/可观测能力不足。
4. **创始人离开**：Peter Steinberger 加入 OpenAI（2026.2.14）→ 项目转基金会治理，存在方向不确定性。

### 8.3 竞争格局五条主线

| 主线 | 描述 | 代表产品 |
|------|------|----------|
| **① 开源本地派** | 强调数据主权、模型自由度、社区驱动 | OpenClaw 原版 / CoPaw / LobsterAI / OpenClaw-CN |
| **② 云端全栈派** | 强调开箱即用、无需技术能力 | Manus / Perplexity Computer / 扣子空间 |
| **③ 国产云托管派** | 面向中国市场，解决翻墙/合规/模型适配问题 | KimiClaw / MaxClaw |
| **④ 大厂嵌入派** | Agent 能力嵌入已有生态（Desktop/Office/IDE） | Claude Cowork / OpenAI Operator / Google Mariner |
| **⑤ 硬件形态派** | 专用 AI 设备 | Rabbit R1（困境中）/ Humane AI Pin（已失败） |

### 8.4 未来 6 个月关键变量

1. **OpenClaw 基金会化能否维持迭代速度**？创始人离开后的治理模式是最大不确定性。
2. **CoPaw 正式开源后能否分流 OpenClaw 中国社区**？阿里云背书 + 极简安装 + 钉钉/飞书原生支持 = 对国内用户有强吸引力。
3. **Manus 被 Meta 收购后的整合方向**？若整合到 WhatsApp/Messenger → 直接威胁 OpenClaw 的 IM 入口优势。
4. **安全合规是否成为行业准入门槛**？谷歌已大规模封禁 OpenClaw 用户 → 第三方平台治理压力将重塑格局。
5. **MCP 协议是否成为行业标准**？Anthropic 提出、OpenClaw/扣子空间等广泛采纳 → 生态锁定效应正在形成。
