## 一、OpenClaw 产品基准（对比锚点）

| 维度        | OpenClaw                                                                                 |
| --------- | ---------------------------------------------------------------------------------------- |
| 定义        | 开源本地优先的个人 AI Agent 平台，通过 IM 应用（WhatsApp/Telegram/Slack/Discord/钉钉/飞书等 50+）交互，能操控电脑执行真实任务 |
| 架构        | 客户端层 → Gateway 控制平面（`ws://127.0.0.1:18789`）→ Pi Agent 执行运行时                              |
| 模型        | 任意 LLM（OpenAI/Claude/Gemini/DeepSeek/Kimi/通义/混元等）                                        |
| Skills 生态 | ClawHub 5,700+ 社区技能                                                                      |
| 记忆        | `MEMORY.md` + 日志式持久记忆                                                                    |
| 协议        | MCP（Model Context Protocol）深度集成                                                          |
| 许可        | MIT License                                                                              |
| GitHub    | 263k+ stars / 50.3k forks（截至 2026.3）                                                     |
| 创始人       | Peter Steinberger（奥地利），2026.2.14 加入 OpenAI，项目转基金会治理                                      |

> 🔗 **官方链接**
> - GitHub：https://github.com/openclaw/openclaw
> - 文档站：https://docs.openclaw.ai
> - ClawHub 技能市场：https://clawhub.ai

## 五、关键趋势与竞争主线总结

### 5.1 五条竞争主线

| 主线 | 描述 | 代表厂商 |
|------|------|----------|

| **③ 云端全栈派** | 开箱即用，无需技术能力 | Manus / Perplexity Computer / 扣子空间 |
| **④ 大厂嵌入派** | Agent 能力嵌入已有生态 | Claude Cowork / OpenAI Operator |
| **⑤ 人才/生态收编派** | 通过收购/招聘控制关键节点 | OpenAI（收编创始人）/ Meta（收购 Manus $25 亿）|

### 5.2 未来 6 个月关键变量

1. **OpenAI × OpenClaw**：Steinberger 加入 OpenAI 后，"下一代个人智能体"何时发布？是否与 OpenClaw 生态深度整合？
2. **Meta × Manus × WhatsApp**：Manus 若整合到 WhatsApp/Messenger → 20 亿+用户基数直接威胁 OpenClaw 的 IM 入口优势。
3. **CoPaw 开源后的社区分流**：阿里云背书 + 极简安装 + 钉钉/飞书原生 = 对国内 OpenClaw 用户有强吸引力。
4. **MCP 协议标准化**：Anthropic 提出 → OpenClaw/扣子空间/OpenAI/Notion 广泛采纳 → 谁能定义 MCP 生态 = 谁控制 Agent 的工具层。
5. **安全合规**：谷歌已大规模封禁 OpenClaw 用户 → 第三方平台治理压力将重塑格局。
6. **MaxClaw 港交所上市后的增长**：MiniMax 2026.2 ARR $1.5 亿 → MaxClaw 是否成为增长引擎？

### 5.3 OpenClaw 的护城河与短板

**护城河**：
- 开源 + 本地优先 = 数据主权（唯一满足此条件且有规模生态的产品）
- 50+ IM 渠道接入 = 基础设施级壁垒
- 5,700+ Skills 飞轮 + 263k GitHub stars 社区势能
- MCP 深度集成 = 工具层标准化先发优势

**短板**：
- 安全漏洞频发（CVE-2026-25253 / ClawHavoc 供应链投毒事件影响 13.5 万台设备）
- 技术门槛高（CLI + Node.js 22 + 配置复杂 → CoPaw/LobsterAI 直接切此痛点）
- 创始人离开 → 基金会治理的方向不确定性
- 合规性差 → 企业级部署受阻
