## 一、三大 IM 入口：微信/QQ/企业微信全线接入 OpenClaw

### 1.1 企业微信：3 步即可接入的官方标准方案

**核心地位**：企业微信是腾讯在 OpenClaw 生态中**首个官方支持**的 IM 平台，也是接入流程最简化的企业级 Agent 通道。

#### 1.1.1 接入方案概览

| 信息项 | 详情 |
|--------|------|
| **官方公告日** | 2026年3月9日 |
| **接入流程** | 3 步：创建机器人 → 获取 Bot ID + Secret → 在 OpenClaw 终端配置 |
| **接入方式** | WebSocket **长连接方式**（无需配置域名、无需公网 IP） |
| **支持能力** | 被动回复 + AI 主动推送消息 + 数据写入智能表格 |
| **兼容版本** | OpenClaw v2026.2.2-3 及以上 |
| **适配云厂商** | 腾讯云 Lighthouse、Kimi、MiniMax、EasyClaw、Coze 等 |

#### 1.1.2 核心功能详解

| 功能 | 说明 |
|------|------|
| **智能机器人对话** | 企业成员直接通过企业微信与 AI 助手对话 |
| **AI 主动推送** | 不仅支持被动回复，还支持 AI 主动向用户发送消息 |
| **智能表格写入** | 通过 Webhook 协议将 OpenClaw 与智能表格打通，数据自动入库 |
| **长连接稳定性** | WebSocket 长链接，断连自动重连，无需域名或 IP 即可接收消息 |

#### 1.1.3 两种接入模式对比

| 能力 | AI 机器人模式 | 自建应用模式 |
|------|-------------|-------------|
| 私聊接收 | ✅ 支持 | ✅ 支持 |
| 流式回复 | ✅ 打字机效果 | ❌ 完整消息回复 |
| 主动发送 | ❌ 不支持 | ✅ 支持（核心优势） |
| 发送文件/图片 | ❌ 受限 | ✅ 完整支持 |
| 消息格式 | JSON 回调 | XML 回调 |
| 适用场景 | 智能对话客服 | 审批通知、定时推送、文件处理 |

#### 1.1.4 战略意义

- **企业市场入口**：企业微信拥有 **2.5 亿活跃用户**，覆盖中国绝大多数大中型企业
- **降低企业 AI 门槛**：管理员仅需 3 步即可完成部署，无需技术背景
- **生态锁定**：企业一旦在企业微信中接入 OpenClaw，工作流和数据均沉淀在腾讯体系内

### 1.3 微信：通过 WorkBuddy/QClaw 实现一键直连

#### 1.3.1 微信接入方式总结

微信由于其封闭生态的特殊性，并非像企业微信/QQ 那样提供官方开放 API，而是通过**间接方式**实现接入：

| 接入路径 | 实现方式 | 安全等级 | 难度 |
|----------|---------|---------|------|
| **WorkBuddy 微信直连**（推荐） | 通过配置微信客服号，扫码绑定 | ⭐⭐⭐⭐⭐ 最高（官方 SDK + WebSocket） | 极低 |
| **QClaw 微信直连** | QClaw 内置一键连微信功能 | ⭐⭐⭐⭐ 高 | 极低 |
| **企业微信 → 个人微信** | 通过企业微信应用消息推送到个人微信 | ⭐⭐⭐⭐ 高（官方 API） | 中等 |
| **Wechat-Claw 第三方插件** | 社区开发的微信适配插件 | ⭐⭐⭐ 中等 | 较高 |

#### 1.3.2 WorkBuddy 微信一键直连（核心方案）

**2026年3月12日**，WorkBuddy 发布全新版本，**微信一键直连**是最大亮点：

- **配置方式**：点击右上角"Claw 设置" → 配置微信客服号 → 扫码绑定
- **使用场景**：通勤、外出等场景下通过微信发送文字指令，电脑上的 WorkBuddy 即时执行
- **7×24 小时**：只要电脑不关机，即可实现随身 AI 助理
- **安全性**：全程本地运行，数据不上传云端

#### 1.3.3 战略意义

> **微信是全球月活最高的超级 App（14 亿 MAU），腾讯通过 WorkBuddy/QClaw 间接打通微信与 OpenClaw，意味着 OpenClaw 的指令入口从"技术极客的终端"直接下沉到"每个人的微信聊天框"——这是其他任何竞争对手都无法复制的独家壁垒。**
