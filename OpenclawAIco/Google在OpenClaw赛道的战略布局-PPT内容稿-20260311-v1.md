## 【左栏】Google 自研 Agent 产品线——全面布局，声量有限

### ▎产品全景：模型→产品→协议，端到端覆盖

| 层级      | 产品/技术                   | 状态                     | 一句话定位                                   |
| ------- | ----------------------- | ---------------------- | --------------------------------------- |
| **模型层** | Gemini 2.5 Computer Use | 2025.10 Public Preview | Google首个界面控制AI模型；13种操作、支持浏览器+移动端        |
| **产品层** | Project Mariner         | 测试中（15个月未开放）           | Chrome扩展Agent；WebVoyager 83.5%；仅限浏览器标签页 |
| **产品层** | Jules                   | 2025年~8月正式上线           | 异步AI编程Agent；GitHub深度集成，"你睡觉它干活"         |
| **产品层** | Agentspace              | 2025.4 正式发布            | 企业级AI Agent平台；跨系统搜索+Agent Gallery+无代码创建 |
| **协议层** | A2A协议                   | 2025.4 发布              | Agent间通信开放标准；50+公司支持，与MCP互补             |

### ▎核心能力亮点

- **Gemini 2.5 Computer Use**
  - 四步循环交互：截图→AI解析→执行操作→捕获新状态→循环
  - 速度优势：第三方实测"比竞品快50%"（Poke.com反馈）
  - 移动端支持：可自定义`open_app`、`long_press_at`等移动操作，差异化优势
  - 内部验证：Google支付平台团队修复超60%的端到端UI测试失败

- **A2A协议**：战略远见最强的一步——不争产品声量，争Agent间通信的**规则制定权**

### ▎关键短板：为什么"什么都做了，没一个冒出来"

| #   | 短板                     | 具体表现                                              |
| --- | ---------------------- | ------------------------------------------------- |
| 1   | **Computer Use仅限浏览器级** | 明确声明"未针对桌面OS级控制优化"，而GPT-5.4已OSWorld 75%超越人类       |
| 2   | **产品化速度慢**             | Mariner从2024.12公布至今15个月仍为"受信任测试者"阶段，"今年上线"承诺迟迟未兑现 |
| 3   | **缺乏爆款时刻**             | 没有Claude Code定义赛道、GPT-5.4一夜翻盘那样的戏剧性事件             |
| 4   | **"追随者"标签**            | 业界普遍认知：Google从AI技术开创者变成了追随者                       |
| 5   | **功能淹没在Gemini大盘**      | CU、Mariner、Jules都是Gemini生态子功能，缺乏独立品牌认知            |
| 6   | **企业优先，消费者感知弱**        | Agentspace面向B端，公众几乎无感                             |

## 【右栏】Google Workspace CLI（gws）——开源基础设施的精准一击

### ▎事件概览

| 维度 | 详情 |
|------|------|
| **名称** | Google Workspace CLI，简称 **gws** |
| **上线时间** | 2026年3月初（GitHub） |
| **组织归属** | Google Workspace 官方GitHub组织名下 |
| **开发语言** | **Rust** |
| **GitHub Stars** | **15,000+**（上线数天） |
| **推广人** | Google Cloud AI总监 **Addy Osmani**，推文**500万+浏览量** |
| **重要声明** | ⚠️ **"非正式产品"**——Not an officially supported Google product |

### ▎核心功能：Google Workspace全家桶统一命令行入口

| 覆盖产品                | 典型操作                                    |
| ------------------- | --------------------------------------- |
| **Gmail**           | 发送/搜索/管理邮件及标签                           |
| **Google Drive**    | 上传/下载/搜索/管理文件及权限                        |
| **Google Calendar** | 创建/修改日程、管理提醒                            |
| **Google Sheets**   | 读写表格数据、创建电子表格                           |
| **Google Docs**     | 创建/编辑文档                                 |
| **Google Chat**     | 发送聊天消息                                  |
| 其他                  | 通过Discovery Service**动态发现、零更新自动适配新API** |

### ▎技术亮点：两阶段动态解析

```
用户/Agent输入命令 → 识别目标服务 → 动态获取Discovery Document
→ 运行时构建命令树 → 执行并返回结构化JSON
```

- **传统CLI**：API硬编码、新API需更新版本、每个服务一个SDK
- **gws**：运行时从Discovery Service动态拉取，**零更新自动适配所有Workspace API**，单一二进制文件覆盖全部服务

### ▎Agent集成能力：为什么被称为"AI Agent的数字手臂"

- 内置**40+ Agent Skills**，AI智能体可直接调用
- 输出**结构化JSON**，Agent可直接解析
- `--dry-run` 预览模式：Agent可先预览再执行，降低误操作
- **自动处理OAuth认证、分页、错误重试**——Agent无需关心底层逻辑

**与OpenClaw一键接入场景**：
> 用户："帮我整理本周所有AI Agent相关邮件，汇总到Google Sheets"
> → OpenClaw调用 `gws gmail messages list` → 解析JSON → 调用 `gws sheets values update` → 完成

### ▎战略解读：为什么此时开源？为什么精准？

| 维度             | 判断                                                               |
| -------------- | ---------------------------------------------------------------- |
| **精准借势**       | OpenClaw 2026年2-3月引爆全球Agent热潮，gws精准卡位这一时间窗口                      |
| **生态卡位**       | 不争Agent本身，争Agent的**数据接口层**——谁的办公数据能被Agent操作，谁就掌握Agent化办公的入口      |
| **护城河**        | Gmail/Drive/Calendar覆盖**30亿+用户**的数据，这是任何竞争对手都无法复制的壁垒             |
| **核心逻辑**       | 即使Google没赢Agent层竞争，只要Agent们都需操作Google数据，Google就是**不可绕过的基础设施提供者** |
| **Chrome时代翻版** | 类比：Chrome免费开源→人人在Chrome上用Google搜索；gws开源→所有Agent通过gws操作Google数据   |

### ▎争议与风险

| 风险 | 说明 |
|------|------|
| **非正式产品** | 明确声明无官方支持，用户自行承担风险 |
| **数据安全** | Agent获得Gmail/Drive全权限，误操作可能导致误删文件、误发邮件 |
| **无SLA** | 企业使用存在合规风险 |
| **快速演进** | API可能随时变更，稳定性待验证 |

### ▎一句话小结

> gws的精准在于：它不跟Anthropic争Agent，不跟OpenAI争模型——它争的是所有Agent都绑不开的**数据基础设施层**，30亿用户的办公数据就是最深的护城河

## 【底部总结区】

### 核心判断

| 维度 | Google自研Agent产品线 | Google Workspace CLI |
|------|---------------------|---------------------|
| **策略** | 全栈覆盖，面面俱到 | 单点突破，精准卡位 |
| **结果** | 声量一般，缺乏爆款 | 上线即爆火，15k Stars |
| **本质** | 争Agent层的话语权（但被Anthropic/OpenAI压制） | 争Agent的数据基础设施层（不可替代性极强） |
| **类比** | Android生态——什么都有但没一个最好 | Chrome浏览器——免费开源但人人离不开 |

### Google最大的缺失

> **缺少一个面向消费者的"杀手级"Agent入口**——Anthropic有OpenClaw，OpenAI有GPT-5.4的病毒式传播，而Google目前没有一个让普通用户"Wow"的Agent产品

### 5个待观察变量

1. Gemini下一版本是否升级至**桌面级Computer Use**（从浏览器级→桌面级）
2. Project Mariner能否在2026年内**兑现面向消费者的正式上线承诺**
3. A2A协议能否与MCP**形成真正的互补生态**并获得广泛采纳
4. gws能否**升级为正式产品**并提供SLA和官方支持
5. Google是否会推出一个**类似OpenClaw的消费级杀手Agent**
